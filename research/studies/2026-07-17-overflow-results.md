# 2026-07-17 通常コメントoverflow観測結果

## 目的

通常 `naka` コメントの可変高スロットが表示領域下端へ達したとき、固定レーンへの循環、先頭への再探索、非表示化、ランダム配置のどれが起きるかを分離する。保存済み公式バンドルから読んだランダムYフォールバックを、独立したオフライン再生で反証する。

## 入力と条件

`research/scenarios/overflow-medium-14.json` のmedium単行14件を、すべて `vposMs=32000` で同一forkへ投入した。同一入力を新しい匿名Chromeプロファイルで3回独立再生し、各runで14件すべてがCanvas描画へ到達したことを `scenarioTextHitCount=14` で確認した。

隔離再生は保存済みレスポンスだけを提供し、NicoCache由来の `/local/` 資産を無効化した。各runには記録外要求が29〜30件あったため `--allow-misses` を使ったが、WebSocket試行は0、scenario応答置換は1、保留中handlerは0だった。未記録要求はすべて遮断され、ネットワークへフォールバックしていない。

## 結果

1〜11番は3回すべて同じYへ配置された。外側Canvasのtranslation Yは先頭が `1.560843 px`、以降は `68.1646 px` 間隔で増え、11番は `683.206848 px` だった。

12〜14番は同じ入力・同じ処理順にもかかわらず、runごとに異なるYへ配置された。

| コメント   |      run 1 |      run 2 |      run 3 |
| ---------- | ---------: | ---------: | ---------: |
| 11（対照） | 683.206848 | 683.206848 | 683.206848 |
| 12         | 220.287567 |  16.934296 | 613.680542 |
| 13         | 446.716980 | 455.377899 | 326.536713 |
| 14         | 575.382813 | 523.520813 | 701.565308 |

処理順は全runで `no` 昇順の1〜14番だった。12〜14番のYは既存の `68.1646 px` 間隔へ量子化されず、run間でも再現しない。run 3の14番は11番の縦区間と重なり得る位置にあり、overflow後に空き区間を再検査していないことも支持する。

## バンドルとの照合

保存済み `PlayerSeekBar-DhFwmJ0e.js`（response body SHA-256 `493fbda6d276b40b6a1cc903a2d867729aace173d61df0843010a2b70d5e9205`）の処理は、候補下端が表示領域下端以上になった時点で通常探索を打ち切り、次の式を返す。

```text
renderArea.top + random() * (renderArea.height - comment.height)
```

実測した末尾3件の非量子化Y、run間変動、既存区間との重なり可能性はすべてこの式と整合する。したがって現時点では、overflow時の一般規則を「表示領域内へ収まる範囲のランダムYを1回選び、衝突を再検査しない」とする。

## CO実装への反映

v4.1.0の `src/renderer/activation.ts` は、通常探索で候補下端が表示高以上になった場合に `Math.random() * (displayHeight - slotHeight)` を採用する。これはbundle読解と今回のブラックボックス観測の両方に一致する。

## 次の反証

1. 横画面とは別定数を持つ縦長・short表示で、文字サイズ、行高、自動縮小閾値を独立観測する。
2. `ue` / `shita` のoverflowでも同じランダムフォールバックと、固定コメント固有の短時間重ね許容条件を分離する。
