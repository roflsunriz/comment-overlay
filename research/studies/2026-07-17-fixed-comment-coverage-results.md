# 2026-07-17 固定コメント未観測範囲の検証結果

## 結論

固定コメントの縦配置は、同一vpos、`full`、`mincho`、`ca` といった個別条件ではなく、表示中の予約区間と新しいコメントの論理slot高だけで説明できる。

1. 上固定は上端から、下固定は下端から、既存予約と重ならない最初の区間へ置く。
2. 新しいslot高が表示高未満で、空き区間だけが足りない場合は、表示範囲内のランダムYへ1回だけ退避する。
3. 新しいslot高が表示高以上の場合は探索せず、上固定は上端、下固定は `表示高 - slot高` へ置く。後続レイヤーも同じ座標になる。
4. 固定表示時間は3000msで、`+2999ms`までは同時表示、`+3000ms`で先行コメントが消える。

第5話EDの矩形、白抜き、歌詞は16行の `ue big full mincho` レイヤーで、論理slot高が768以上になる。したがって全レイヤーが同じ上端へ重なり、作品固有・`ca`固有の分岐は不要である。

## 実験方法

- 入力生成: `research/tools/run-fixed-comment-matrix.mjs`
- 教師: 保存済み公式プレイヤー資産の外部通信遮断リプレイ
- 観測: source canvas寸法、outer canvasの変換行列、処理sequence、video時刻
- 生データ: `research/runs/fixed-comment-matrix-*`（Git管理外）
- 個別レイヤー対照: `research/scenarios/fixed-full-mincho-layered-ca.json`

同一条件を別Chromeプロセスで反復し、決定的な端固定とランダム退避を区別した。

### 実測した代表値

outer canvas高768に対する上固定2層のtranslation Yは次のとおりだった。

| 条件       |           Y1 |      Y2 | 分類                         |
| ---------- | -----------: | ------: | ---------------------------- |
| big 1行    |       -0.674 |  97.988 | 整列                         |
| big 15行   | 表示上端付近 | run依存 | 空き不足のランダム退避       |
| big 16行   |       -1.340 |  -1.340 | 表示高以上の端重畳           |
| big 30行   |       -2.063 |  -2.063 | 表示高以上の端重畳           |
| small 30行 |       -2.385 | run依存 | 単体は収まるためランダム退避 |

下固定big 16行は両層とも `-1.615`、big 30行は両層とも `-670.892` だった。source texture由来の小数オフセットを除けば、それぞれ `表示高 - slot高` の同一端である。

## 横幅の補足

big/mincho単行では、10文字まではsource font 39px・縦間隔98.662pxだった。30文字では17px・43.287px、60文字では10pxに0.8 scale・21.744px、120文字では10pxに0.4 scale・10.972pxへ縮小した。12 tabは公式側でem spaceへ正規化され、20px・51.905pxになった。

この縮小は縦overflow規則とは独立している。今回のコメントアートは行数起因の縮小後でもslot高が768以上になるため、端重畳の修正は横幅用の個別条件を使わない。tab正規化はCanvas解析器にも反映し、既存traceを本文へ正しく対応付けられるようにした。

## サンドボックス上の制約

異なるpositionを同一合成応答へ入れたinteraction profileは、公式プレイヤーのheadless描画プロセスが高負荷のまま終了しない状態になった。一時プロファイルをOS一時領域と `research/` 配下の双方で再現したため、得られなかった座標を推測値で埋めていない。`ue` と `shita` は個別profileで同じ境界規則を確認済みであり、実装でも予約集合を分離したままにした。この停止現象自体は公式配置の根拠として扱わない。

## `src/` への反映

固定配置を次の純粋な決定として実装した。

```text
if slotHeight >= displayHeight:
    y = 0                         # ue
    y = displayHeight-slotHeight  # shita
else if edgeから探索した空き区間がある:
    y = 最初の空き区間
else:
    y = random() * (displayHeight-slotHeight)
```

通常整列、空き不足、表示高以上、期限切れ予約、実CA全11バッチを自動テストで覆った。固定コメントを画面外へ無制限に積み上げていた旧処理は除去した。広幅単行コメントの高さ補正は独立した既存規則として維持し、今回の多行CA境界とは混ぜていない。
