# コントリビューションガイド

`comment-overlay`への貢献に興味を持っていただきありがとうございます！
このプロジェクトは、コミュニティからのバグ報告、機能提案、Pull Requestを歓迎します。

## バグ報告

バグを発見した場合は、以下の手順で報告をお願いします。

1.  まず、既存のIssuesを検索し、同様の報告がないか確認してください。
2.  同様の報告がない場合は、バグ報告用のテンプレートを使用して新しいIssueを作成してください。

Issueには、バグを再現するための具体的な手順、期待される動作、実際の動作、そしてお使いの環境（OS、ブラウザー、Node.jsのバージョンなど）をできるだけ詳しく記述してください。

## 機能提案

新しい機能や改善案を提案したい場合は、機能提案用のテンプレートを使用してIssueを作成してください。

提案には、どのような問題を解決したいのか、なぜその機能が必要なのかを具体的に説明してください。

## Pull Request (PR) の送り方

コードの修正や機能追加はPull Requestを通じて受け付けています。以下の手順に従ってください。

### 1. 開発環境のセットアップ

まず、このリポジトリをフォークし、ローカルにクローンします。

```bash
# 依存パッケージをインストール
npm install
```

### 2. ブランチの作成

変更内容に応じたわかりやすい名前のブランチを作成します。

```bash
# 例: fix-rendering-bug という名前のブランチを作成
git checkout -b fix-rendering-bug
```

### 3. コードの変更と動作確認

コードを修正・追加します。開発中は以下のコマンドが役立ちます。

```bash
# Vite 開発サーバーを起動
npm run dev
```

変更後は、ローカルでビルドし、サンプルページで動作確認を行ってください。

```bash
# ライブラリをビルド
npm run build

# サンプルページを起動して動作確認
npm run serve
```

### 4. 品質チェック

Pull Requestを提出する前に、必ず以下のコマンドを実行して、コードの品質に問題がないことを確認してください。

```bash
# ESLintでコードを検査
npm run lint

# 型チェックを実行
npm run type-check
```

### 5. Pull Requestの提出

変更内容をコミットし、フォークしたリポジトリにプッシュします。
その後、GitHub上でPull Requestを作成してください。PRのテンプレートに従い、変更の概要や関連するIssue番号を記述してください。

あなたのコントリビューションをお待ちしています！
